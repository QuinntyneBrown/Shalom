# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Shalom is a single-user personal wellness PWA (iPhone-first) weaving together morning faith rhythms (Bible reading, prayer, check-in), health (16:8 intermittent fasting, treadmill/bike/elliptical workouts), and intentional connection with family and friends. The approved build plan and UX spec live in the owner's plan file; locked decisions are recorded as ADRs in `docs/adr/` ‚Äî read the relevant ADR before changing the area it describes.

**The code organization deliberately mirrors `C:\projects\saturdaze`.** When a pattern question comes up, saturdaze is the precedent (its `CLAUDE.md` and `docs/adr/` apply by analogy).

## Repository layout

Three sibling top-level directories make up the application; `e2e` is NOT inside `frontend`:

- `backend/` ‚Äî .NET 10 Clean Architecture solution (`Shalom.sln`)
- `frontend/` ‚Äî Angular 21 workspace with three projects: `shalom` (app), `api` (lib), `components` (lib)
- `e2e/` ‚Äî Playwright suite, Page Object Model (POMs in `pages/`, specs in `tests/`, fixtures in `fixtures/`); POM infrastructure generated by the `ppg` dotnet tool (`ppg workspace ./frontend -o ./e2e`)
- `design/shalom.pen` ‚Äî the authoritative UI design (ADR-003); access only via Pencil MCP tools, never Read/Grep
- `docs/adr/` ‚Äî architecture decision records
- `scripts/Start-FreshStack.ps1` ‚Äî one-command fresh stack (pack CLI ‚Üí reset DB ‚Üí build ‚Üí run both processes)

## Common commands

```powershell
# Full local stack from a clean DB (pwsh)
powershell .\scripts\Start-FreshStack.ps1

# Backend
dotnet build  .\backend\Shalom.sln
dotnet test   .\backend\Shalom.sln
dotnet run --project .\backend\src\Shalom.Api --urls http://localhost:5100

# CLI (database lifecycle ‚Äî the API does NOT migrate on startup)
$env:SHALOM_CONNECTION = "Server=(localdb)\MSSQLLocalDB;Database=Shalom;Trusted_Connection=True;TrustServerCertificate=True"
dotnet run --project .\backend\src\Shalom.Cli -- migrate
dotnet run --project .\backend\src\Shalom.Cli -- seed
dotnet run --project .\backend\src\Shalom.Cli -- reset --yes

# Frontend (from frontend/)
npm start                                       # ng serve on :4200
npm run build -- shalom --configuration development
npm test                                        # Vitest via @angular/build:unit-test

# E2E (from e2e/, once it exists ‚Äî requires dev server on :4200)
npx playwright test
```

API runs on `http://localhost:5100` (Swagger at `/swagger`); the Angular environment compiles this URL in.

## Backend architecture (.NET 10 Clean Architecture)

`Directory.Build.props` enforces `TreatWarningsAsErrors`, `net10.0`, nullable on. Central package management in `Directory.Packages.props`. Layer dependency direction (strict):

- `Shalom.Domain` ‚Äî entities + enums, zero deps
- `Shalom.Application` ‚Äî MediatR handlers (feature folders: CheckIns, Reading, Fasting, Workouts, People, Gratitude, Today, Auth), validators, DTO contracts, `IAppDbContext`, `StreakCalculator`
- `Shalom.Infrastructure` ‚Äî `AppDbContext`, EF migrations, SQL Server, auth services, seeder
- `Shalom.Api` ‚Äî controllers, `Program.cs`, DI composition, Swagger, middleware
- `Shalom.Cli` ‚Äî packaged dotnet tool (`shalom migrate|seed|reset`)

Rules that aren't obvious from the code:

- Business rules live in handlers/domain services, not controllers.
- The API does **not** apply EF migrations on startup. Run `shalom migrate` explicitly.
- Seed data is idempotent and safe to re-run.
- Auth is **local JWT only** (bcrypt + SQL) ‚Äî single user, no social/OAuth.
- Every "which day is this" fact is a `DateOnly` derived **server-side** in `America/Toronto` via `IClock` ‚Äî clients never send "today" (ADR-004). Timestamps are UTC `DateTimeOffset`.
- Streaks and connection nudges are **derived at read time, never stored** (ADR-004).
- Fasting is **schedule-first** (ADR-005): explicit `FastingSession` rows, outcome derived in DTO, at most one open session.
- `GET /api/today` is the aggregate powering the dashboard in one round trip and is the single offline-cached resource.

### Tests

All in `backend/tests/`: `Shalom.Application.Tests` (pure unit, parallel), `Shalom.Infrastructure.Tests` (real LocalDB), `Shalom.Api.Tests` (`WebApplicationFactory<Program>`, sequential), `Shalom.Cli.Tests`.

LocalDB on this machine connects via **named pipe**, not the `(localdb)\Instance` shortcut (SQL Server 2025 / SqlClient interop bug) ‚Äî `Support/LocalDbConnection.cs` resolves the pipe via `sqllocaldb info`; `Start-FreshStack.ps1` does the same. See saturdaze ADR-001.

Smart App Control is On on this machine: freshly built DLLs loaded via reflection can be OS-blocked (`0x800711C7`). Don't rebuild the globally installed `ppg` tool; pack-and-install flows (the CLI in `Start-FreshStack.ps1`) follow saturdaze's approach.

## Frontend architecture (Angular 21)

| Project | Type | Purpose |
| --- | --- | --- |
| `shalom` | application | Routed pages, ritual flow, dialogs, guards, interceptor, `TodayStore` |
| `components` | library | Standalone UI components; selectors use `sh-*` |
| `api` | library | DTOs, service contracts + concrete services |

### Conventions enforced across the codebase

- **Selectors stay `sh-foo`**, but TypeScript class names, folders, and files **drop the prefix** ‚Äî e.g. `components/src/lib/bottom-nav/bottom-nav.ts` exports `class BottomNav` with `selector: 'sh-bottom-nav'`.
- **Interface-driven services**: every `api` service pairs a `*.service.contract.ts` (interface + `InjectionToken`) with the concrete class. Pages inject the token (e.g. `AUTH_SERVICE`), never the class. This pattern is load-bearing ‚Äî `ppg bridge` generates e2e service mocks from it.
- **No inline forms in pages.** Button-triggered editing opens a CDK Dialog/bottom sheet or navigates.
- **All modals use `@angular/cdk` Dialog/Overlay** ‚Äî never hand-roll a modal.
- State is Angular Signals; no NgRx. `TodayStore` loads `GET /api/today`, exposes `computed()` per card, applies optimistic patches, and refetches on `visibilitychange` when the cached local date is stale (iOS resumes PWAs days later ‚Äî never trust an in-memory "today").
- **Bottom-nav iOS chrome handling**: `sh-bottom-nav` clears iOS Safari chrome via `--sh-chrome-bottom`, written by a `VisualViewport` listener registered in `main.ts` *before* `bootstrapApplication`. Don't simplify ‚Äî see saturdaze ADR-005.
- Design tokens come from `design/shalom.pen` (ADR-003): cedar teal `#2D5F5D`, warm sand `#C8A57B`, shalom gold `#B8924A` (completion moments only), Fraunces for scripture/display, Figtree for body, warm off-white surfaces, pre-sunrise dawn theme.

### UX rules (the product's soul ‚Äî do not violate)

- Progressive disclosure: day-1 surface is 3 cards; features are *surfaced* via one Discovery card at a time, never locked, never two days in a row.
- Grace-forward tone everywhere: no guilt copy, no red "overdue", no broken-chain imagery. Streaks have Sabbath grace (one missed day/week doesn't break them) and never apply to relationships.
- One connection nudge per day max; none if he already connected with someone today.
- Night state (22:00+) has no CTAs. The app models rest, not hustle.

## Deployment

GitHub Actions: publish API to Azure App Service, run `shalom migrate`, build Angular, deploy to Azure Static Web Apps. Resource group `shalom-rg` (canadacentral). See ADR-002. Secrets live in `.deploy/azure.env` locally (gitignored) and as GitHub Actions secrets.
